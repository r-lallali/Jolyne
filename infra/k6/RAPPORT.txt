================================================================================
 3. POINT DE RUPTURE
================================================================================
 Seuil : entre 6000 et 7000 connexions WS concurrentes (~3000-3500 conv).

   - <= 6000 conn : sain. Connexion ~100 ms, msg_rtt p95 26 ms, p99 49 ms.
                    Premiers signes de tension : rest_latency p95 21 -> 127 ms
                    et healthz qui monte (voir §4).
   - 6000 -> 7000 : effondrement. ws_connecting passe de ~100 ms à 43 s puis
                    60 s (timeout) ; plus aucune nouvelle socket acceptée.
   - 7000 -> 8000 : backend injoignable, puis HTTP 503/404 (Traefik sans
                    backend sain) et connection refused -> le conteneur gateway
                    est tombé et a redémarré.

================================================================================
 5. CONCLUSIONS & RECOMMANDATIONS
================================================================================
 - Objectif "1000 conversations simultanées" : ATTEINT avec ~6x de marge.
   Latence message p95 17 ms / p99 34 ms, matching instantané, HTTP intact.
 - Plafond stable mesuré : ~6000 connexions (~3000 conversations). Au-delà de
   ~6500-7000, le gateway sature et le conteneur redémarre (panne ~108 s,
   auto-récupérée).
 - Plafond "confort" recommandé : ~5000 connexions — au-delà, la latence HTTP
   commence à se dégrader (rest p95 127 ms à 6000 vs 21 ms à 2000).

 Cause probable : croissance mémoire/goroutines (chaque WS = plusieurs
 goroutines + buffers + room Redis). À ~6-7k connexions le conteneur a
 vraisemblablement atteint sa limite mémoire (OOM-kill) puis redémarré.
 À confirmer côté Dokploy :
   docker inspect <backend> --format '{{.State.OOMKilled}} {{.HostConfig.Memory}}'
   docker logs <backend> --since 30m | tail

 Leviers pour repousser le plafond :
   - augmenter la limite mémoire du conteneur backend (Dokploy) ;
   - répliquer le gateway en N instances derrière Traefik (state dans Redis =>
     scale horizontal sans refactor) ;
   - poser GOMEMLIMIT / GOMAXPROCS adaptés au conteneur.
